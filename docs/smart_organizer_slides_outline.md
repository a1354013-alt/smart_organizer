## Slide 3: 核心優化 (1 & 2) - 視覺化與檢索革命 (UX & Data)

| 項目 | 內容 | 視覺建議 |
| :--- | :--- | :--- |
| **標題** | **V2 核心優化 (一)：視覺化預覽與全文檢索** | |
| **優化 1：視覺化預覽 (UX Revolution)** | **問題**：使用者在確認分類時，需要手動打開檔案。 **解決方案**：整合 `pdf2image` 與 `Pillow`。PDF 自動轉為首頁縮圖，照片直接顯示。 **效益**：將「預覽與確認」流程效率提升 **80%**，大幅降低人工校正的認知負擔。 | 介面截圖：展示「預覽與確認」分頁，左側是縮圖，右側是可編輯的標籤。 |
| **優化 2：全文檢索 (Data Revolution)** | **問題**：檔案內容無法被搜尋，價值被鎖定。 **技術選型**：採用 **SQLite FTS5** 模組。在處理時將提取的文字內容存入 FTS 虛擬表。 **效益**：實現**秒級**內容檢索。使用者可搜尋「統編 12345678」或「某個關鍵條款」，系統直接返回相關檔案與匹配片段。 | 介面截圖：展示「全文檢索」分頁，顯示搜尋結果與 FTS5 的 `snippet()` 函數高亮片段。 |

## Slide 4: 核心優化 (3) - LLM 智慧摘要 (AI Intelligence)

| 項目 | 內容 | 視覺建議 |
| :--- | :--- | :--- |
| **標題** | **V2 核心優化 (二)：LLM 智慧摘要與精準分類** | 簡潔的流程圖：PDF/JPG -> Core.py -> LLM API -> JSON Output |
| **問題與動機** | 傳統 Rule-based 分類無法處理語義複雜度，且無法為檔案提供有意義的描述。 | |
| **解決方案** | **整合 OpenAI API (GPT-4o-mini)**：將文件文字內容傳給 LLM。 **LLM 任務**：1. 生成 50 字以內的**簡短摘要**。 2. 建議 **3 個高相關度標籤**。 **效益**：
    *   **分類準確度提升**：AI 建議標籤作為人工校正的強大輔助。
    *   **數據價值最大化**：摘要欄位讓使用者不需打開檔案即可理解內容。 | 介面截圖：展示 AI 摘要的結果，強調摘要的精煉與標籤的準確性。 |
| **架構考量** | **成本控制**：使用 `gpt-4o-mini` 確保低延遲與低成本。 **可擴展性**：架構已預留，未來可輕鬆替換為其他 LLM 或本地模型。 | |
